# Roadmap

This roadmap sequences the work to establish and grow the **Hassan Mubiru — Creator of StreetJS** brand across the profile and flagship projects. Each horizon lists prioritized tasks (High → Medium → Low), expected outcomes, and milestones.

## 30-day — Foundation

| Priority | Task |
|---|---|
| High | Publish the profile README to `hassanmubiru/hassanmubiru` with all widgets wired to the brand palette. |
| High | Create the StreetJS repository and pin it at position 1 with a hero README. |
| High | Apply repository descriptions and topics to StreetJS, Vibra, Afrchat, and SCMINER. |
| Medium | Add the banner and Open Graph image to the profile and StreetJS repo. |
| Low | Reserve the remaining pinned slots and confirm pin ordering. |

**Expected outcomes:** a coherent founder identity is visible on the profile; StreetJS appears as the position-1 flagship.
**Milestones:** profile README live; StreetJS repository created and pinned.

## 60-day — Flagship build-out

| Priority | Task |
|---|---|
| High | Flesh out StreetJS: creator narrative, mission, value proposition, description, topics, and badges. |
| High | Publish reproducible StreetJS benchmarks against Express, NestJS, and Fastify. |
| Medium | Apply the reusable README template across all pinned repositories. |
| Medium | Standardize descriptions and topics across all current repositories. |
| Low | Add social preview images to each flagship repository. |

**Expected outcomes:** StreetJS reads as a real, in-development framework with a clear value proposition.
**Milestones:** StreetJS complete with docs and benchmarks; starter template published.

## 90-day — Authority and readiness

| Priority | Task |
|---|---|
| High | Stand up OSS authority: issue templates, contribution guide, and release/changelog strategy. |
| High | Complete the six-slot pinned layout with pre-pinning improvements. |
| Medium | Build out recruiter-facing leadership, architecture, and impact sections. |
| Medium | Set up a project board and enable Discussions on StreetJS. |
| Low | Publish documentation recommendations for StreetJS. |

**Expected outcomes:** active maintainer signals; a complete, recruiter-ready story.
**Milestones:** all six pins filled; first tagged StreetJS release.

## 6-month — Community growth

| Priority | Task |
|---|---|
| High | Drive StreetJS adoption: triage issues, review PRs, ship a regular release cadence. |
| High | Replace placeholder impact metrics with real numbers as data accrues. |
| Medium | Publish technical deep-dive content that links back to StreetJS. |
| Medium | Expand StreetJS docs into a versioned documentation site. |
| Low | Refresh stats widgets and pin ordering based on traction. |

**Expected outcomes:** sustained contribution activity and a growing contributor base.
**Milestones:** first external contributor merged; versioned docs site published.

## 1-year — Thought leadership

| Priority | Task |
|---|---|
| High | Establish StreetJS as a credible framework choice: stable major release and documented adoption. |
| Medium | Position for backend leadership, developer advocacy, and founder opportunities. |
| Medium | Run a recurring content and community cadence. |
| Low | Conduct an annual brand audit and refresh copy, visuals, and pins. |

**Expected outcomes:** durable, self-reinforcing brand recognized for the Creator-of-StreetJS identity.
**Milestones:** StreetJS reaches a stable 1.0; annual re-audit completed.
