## 🚀 StreetJS

**StreetJS** is a modern TypeScript backend framework for building fast, type-safe Node.js APIs and services. It pairs an ergonomic developer experience with end-to-end type safety — typed routing, typed configuration, request/response validation, and a generated client — so teams ship correct services with less boilerplate.

<p>
  <img src="https://img.shields.io/badge/StreetJS-Framework-8B5CF6?style=for-the-badge&logo=typescript&logoColor=F8FAFC" alt="StreetJS" />
  <img src="https://img.shields.io/badge/TypeScript-first-2563EB?style=for-the-badge&logo=typescript&logoColor=F8FAFC" alt="TypeScript-first" />
  <img src="https://img.shields.io/badge/status-in%20development-06B6D4?style=for-the-badge" alt="status: in development" />
  <img src="https://img.shields.io/badge/license-MIT-8B5CF6?style=for-the-badge" alt="MIT" />
</p>

**Why StreetJS**

- **Type-safe by default** — route handlers, params, and config are fully typed; mistakes are caught at compile time.
- **Ergonomic DX** — a minimal, predictable API with first-class TypeScript inference and zero ceremony.
- **Performance-minded** — a lean core and an explicit middleware pipeline built for throughput.
- **Contract-driven** — OpenAPI-first routes and a generated client keep services and consumers in sync.

> StreetJS is under active development. Releases follow the pre-1.0 `0.y.z` line until the API stabilizes for 1.0.
