# Pinned Repositories

A recommendation for the six pinned repositories on [github.com/hassanmubiru](https://github.com/hassanmubiru), in display order. StreetJS is fixed at position 1 as the flagship. Each entry lists the improvements to apply before pinning.

| Position | Repository | Why it is pinned | Improvements before pinning |
|---|---|---|---|
| 1 | **StreetJS** | Flagship — Modern TypeScript Backend Framework; anchors the entire brand. | Hero README opening on the headline; description ≤350 chars; 5–20 topics; ≥4 badges; quickstart; comparison vs Express/NestJS/Fastify. |
| 2 | **streetjs-starter** | Official scaffold; converts interest into a running app fast. | One-command scaffold; passing CI; README using the standard template; link back to StreetJS. |
| 3 | **Vibra** | Showcases real-time product engineering on a TypeScript backend with Flutter clients. | Add architecture diagram; screenshots/GIF; standardized description and topics. |
| 4 | **Afrchat** | Demonstrates real-time messaging, presence, and media at scale. | Feature list; demo media; standardized README; topics covering realtime/chat. |
| 5 | **SCMINER** | Highlights data-mining and analysis depth beyond web backends. | Usage examples; sample dataset/output; clarified scope in description and topics. |
| 6 | **api-forge** | Reinforces the API Architect pillar with an OpenAPI-first toolkit. | OpenAPI example; generated-client demo; badges; standardized README. |

## Notes

- **StreetJS at position 1** is non-negotiable; it is the brand anchor.
- Positions 2–6 balance **breadth** (framework, real-time apps, data tooling, API tooling) with **depth** (each repo demonstrates a distinct Positioning Pillar).
- If a repository does not yet exist (for example `streetjs-starter` or `api-forge`), create it as a candidate so all six slots are filled, then iterate.
- Re-evaluate ordering quarterly based on traction (stars, traffic, and contributions).
